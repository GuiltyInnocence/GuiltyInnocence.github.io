运用GAL云存档练习GIT的使用

新建仓库

比较生疏，所以选择首先云端新建

本地GAL大目录重命名一下，改成英文字符如Floral_flowlove
由于存档本来已经摆放好，因此直接git init
我们只需要同步存档，而不需要管别的游戏文件
因此自然想到直接git add savedata\
（什么？您想把github当网盘使？那随便）
这样做貌似有时会出bug，偶尔会提升untracked files present，一通操作又好了，很玄学
保险的做法是touch .gitignore然后git add .gitignore

设置上游

git remote add origin
git branch --set-upstream-to=origin/master

开始同步

git add savedata/
（如果是BGI还要add bgi.gdb）
git commit -m "balabala"
然后是最愉快的git push origin

设备间同步

笔记本？台式？平板？
No problem!

  • 每次结束游戏时记得git push一下
  • 不然的话……

    冲突合并

    git stash在没有add之前可以放弃更改
    git merge手动选择存档
    （krkr的config.sav``global.sav``read.sav需要手动操作一下；BGI的bgi.gdb同理）
    想要回退存档版本时git reset HEAD^

剧本 9.0 of 10//暂评
  • Part1 于パトリシア线中途 191230
    玩到现在有这些感觉吧。
    首先是剧本的不合常理逻辑部分。
    内因很明显，就是角色的性格特点和设定特征。回不了家的孩子，不想回家的孩子，个个个性鲜明，大家都自说自话。再加上死者的世界来的パト完全不知人间常识，会碰撞出这种跨服聊天神仙对话一样的剧场也不奇怪。
    外因就是电波了，反正我是没怎么对上。这部的脚本经常在奇怪的地方切Chapter，感觉戛然而止一样
    然而这部作品更大的意义在以下
    本作以创作出来的所谓死者的视角观察生活，这样的确会发现许多生活中我们习以为常，不被察觉的东西/细节，特别是长大成人被常识和逻辑束缚认为理所当然的东西。这种观察和思考能让我们更好的感受生活，并去热爱生活。这和小孩子的好奇心、观察力、想象力，以及天真无邪是一样的道理。它促使我们去思考的习以为常的东西，恰恰是本原的、原初的，又是颇具思考价值的。
    重点的来了。在上面所述的思维飞天剧本的掩盖下，这种本原的思考便显得顺其自然，毫无违和感，自然地流淌出来。
